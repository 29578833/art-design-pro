# 系统通知接口目录

> 前端 request 前缀：`/setting`（adminapi）

## 渠道状态约定

| 值  | 含义                                 |
| --- | ------------------------------------ |
| 0   | 关闭                                 |
| 1   | 开启                                 |
| 2   | 该通知不支持此渠道（前端不展示开关） |

常用渠道字段：`is_system`（站内）、`is_sms`（短信）、`is_wechat`（公众号）、`is_routine`（小程序）、`is_app`、`is_ent_wechat`（企业微信）。

## 1. 系统通知列表

- **方法**：`GET`
- **路径**：`/setting/notification/index`
- **参数**：`type`（必填分类：`1` / `2` / `3`）
- **前端约定**：分别请求 `type=1`、`type=2`、`type=3` 三次，将响应数组合并为一份列表展示

### 响应 data

数组列表（非分页），项含：`id`、`name`、`title`、`mark`、`type`、各 `is_*` 渠道字段、`system_title`、`system_text` 等。

## 2. 获取单条通知详情

- **方法**：`GET`
- **路径**：`/setting/notification/info`
- **参数**：`id`（必填）、`type`（渠道类型，如 `is_system`）

按 `type` 回填对应渠道内容字段（如站内信将 `system_text` 映射到 `content`）。

## 3. 保存通知设置

- **方法**：`POST` / `ANY`
- **路径**：`/setting/notification/save`
- **主要参数**：`id`、`type`、`name`、`title`、各渠道开关与模板字段

当 `type=is_system` 时保存：`name`、`title`、`is_system`、`is_app`、`system_title`、`system_text`。

## 4. 修改渠道开关状态

- **方法**：`PUT`
- **路径**：`/setting/notification/set_status/:type/:status/:id`
- **示例**：`/setting/notification/set_status/is_sms/1/12`
- **说明**：将指定通知的某个渠道字段更新为 `status`（0/1）
