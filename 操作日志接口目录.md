# 操作日志接口目录

> 前端 request 前缀：`/system`（adminapi）

## 1. 操作日志列表

- **方法**：`GET`
- **路径**：`/system/log`
- **说明**：管理员后台操作记录分页列表

### 请求参数

| 参数     | 类型          | 说明                                                          |
| -------- | ------------- | ------------------------------------------------------------- |
| page     | number        | 页码，默认 1                                                  |
| limit    | number        | 每页条数，默认 15                                             |
| pages    | string        | 页面名称关键字（模糊匹配 `page`）                             |
| path     | string        | 访问路径关键字（模糊匹配 `path`）                             |
| ip       | string        | IP 关键字                                                     |
| admin_id | number/string | 管理员 ID；不传则按当前账号 level 过滤可见管理员              |
| data     | string        | 时间范围，后端映射为 `time`，格式如 `2026/07/01 - 2026/07/13` |

### 响应 data

```json
{
  "list": [
    {
      "id": 1,
      "admin_id": 1,
      "admin_name": "admin",
      "page": "用户账户",
      "path": "setting/admin",
      "path_name": "管理员列表",
      "ip": "127.0.0.1",
      "type": "GET",
      "add_time": 1720000000
    }
  ],
  "count": 100
}
```

## 2. 操作日志管理员筛选项

- **方法**：`GET`
- **路径**：`/system/log/search_admin`
- **说明**：获取可选管理员列表（受当前账号 level 限制）

### 响应 data

```json
{
  "info": [{ "id": 1, "real_name": "超级管理员" }]
}
```
